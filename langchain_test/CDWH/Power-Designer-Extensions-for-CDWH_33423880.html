DataWarehouse : Power Designer Extensions for CDWH

Created by Michael Hadorn, last modified by Fabian Franzeck on Juni 08, 2022

PowerDesigner maintains the code templates for generating DDL statemnets for a specific target database in so called DBMS files. These files are located in the PD installation directory in ~\Resource Files\DBMS. The HANA 2.0 DBMS file is called HanaDB20.xdb. We made a copy of this file and added some extensions specific for the CDWH (see below).

The DBMS file is an attribute of the model:

This DBMS file will be saved and versioned in the PD repository. You can still keep a local copy of this file on your local VD, but there is actually no need for it (see Working with the PowerDesigner Repository]

We have added 3 extensions to the HANA 2.0 DBMS file:

1. drop table statement replaced:

2. Table properties: new attributes BK column and Delta Load Date Colum

  • You need to specify the business key (BK) column of this table. This relates to the previous BK index in the Oracle CDWH.

  • For delta loads you can specify a column which will define the date range to be loaded

  • If this attribute is set the CDWH library generator will generate code for a delta load (using the specified column as filter criteria)
  • If this attribute is not set the CDWH library generator will generate code for a full load

These two fields will generate table annotations in HANA which will be used as input for the CDWH library generator, i.e.:

3. Column properties: new attribute Join Check Parent Table:

For join checks you need to specify a parent table. The join check will join this column with the BK column of the parent table. The CDWH library generator will generate these join checks accordingly.
