DataWarehouse : Employee onboarding

Created by Fabian Franzeck, last modified on März 21, 2023

This is a step-by-step manual to prepare the environment for a new CDWH developer or employee.


  • 2-4 Weeks (latest!) before first day at work
  • One Week before (now we should have the mail account)
  • Todo: First day at work
  • SAP BO BI Access
  • Setup office printer
  • Software installation

2-4 Weeks (latest!) before first day at work

  1. Official USB Checklist: https://intranet.uhbs.ch/fileadmin/user_upload/3_personal/fuehren_am_usb/checkliste_onboarding.pdf
  2. Get HR paperwork done:
    • Regular USB employees or Fondmitarbeiter: Have a contract issued at Humanresources via Bram
    • "Mitarbeiter ohne Lohn"/External employees (like ITLogix): Fill in an application form "Mitarbeiter ohne Lohn" (Mitarbeiter ohne Lohn (Formular) on this Site (document link might change).
      • Applicant has to fill in: Angaben zur Person (S.1), Vertraulichkeitserklärung (S.4), provide Beilagen:
        • Lebenslauf
        • Kopie Pass oder ID
        • Kopie vorhandene Bewilligung (z. B. C, B, G, L)
        • Diplome (bei Gastärzten/innen, Hospitant/innen, wissenschaftlichen Mitarbeitenden und Doktoranden/innen)
        • alle benötigten Unterlagen bei einer Person aus Drittstaaten (siehe Bewilligung)
      • Funktionsbezeichnung für Badge= "Berater"
      • Phone, PEP Zeiterfassung, Zeiterfassung mit Badge = Nein
      • Arbeitszeitmodell, Pensum, "Name Planblatt" can be left blank
      • Ansprechpartner = Bram Stieltjes, 85175
      • have details regarding Kostenstelle, etc. filled by Bram, let him sign and send to HR. On the form, you have the option to order an Email/AD Login (will be sent to new employee by HR) and a ID-batch.
  3. Fill in a Schlüsselantrag, have it stamped by Bram and send to Schlüsselzentrale
  4. Organize a desk/workstation: New hardware e.g laptop needed (consider delivery times...)?

One Week before (now we should have the mail account)

  1. Add User to Confluence: (So the person can update all wiki pages if something is not working anymore)
  2. Add User to Team Chats in Microsoft Teams (Room: CDWH-developers)
  3. Forward fixed weekly meeting appointments in outlook to new employee
  4. Add personal User to SAP Hana database instances (via SAP Admin, e.g. Thierry Doppler), process see here
  5. Add User to Sharepoint (https://isp.usb.ch/teams/0219/), see also Sharepoint install
  6. Add User for access to L:\DICT\FuAS\CDWH , currently needs to be done with email to Doelfs Peter <peter.doelfs@usb.ch>
  7. Add Email to Email-Group (needs to be actively done for people with Fonds-Verträgen):
    1. Alle_Ressort_D+ICT <Alle_Ressort_ICT@usb.ch>  (used for general communications from ICT) → Barbara Renner can manage this
    2. _ICT CDWH-Team <_ICTCDWH-Team@usb.ch>

Todo: First day at work

  1. Introduction to the team and tour through office
  2. Get the office key and employee batch at Schlüsselzentrale (opening times in intranet)
  3. SOS Helpdesktickets necessary:
    • Access for I:\Verwaltung\ICT_CDWH (e.g. for keepass file)
    • Access to Emailfolder as described here: ICT-MSCCMailbox
    • Access to cdwh@usb.ch: Team Mailbox inkl. Prefect Mails
  4. Fill in Antragsformular: type "antrag" in browser (internet explorer always works)
    • Order Access for relevant source system applications
    • Local admin rights for the Virtualdesktop (or Laptop)
  5. Add User to gitlab group cdwh as owner or maintainer
  6. Follow rest of this tutorial

We work on virtual desktops in USB. It is also possible to access the VDI from outside USB, the manual to setup this functionality can be found here: Externes Login USB

SAP BO BI Access

If required, new colleagues get access to the SAP BO BI Plattform.

  1. Add Windows AD User to the group ICT_B_SAPBO_Admins or ICT_B_SAPBO_Users, depending on their role
  2. Manually Update the Windows AD Group mapping of the SAP BO BI Plattform via Central Management Console on DEV, KONS & PROD
    1. In the CMC: "Authentification" → "Windows AD" → "Update"
    2. For ICT_B_SAPBO_Admins nothing further to do
    3. For ICT_B_SAPBO_Users, add the User to the group: "DWH_Administrator_Designer":

  3. For SAP BO BI Plattform Administrator, the User must request an "A_XXXXX" Account via ITSM Ticket.
    1. Afterwards, the "A_XXXXX" Account must be added to the Windows AD group: "ICT_B_CrystalReports_Admins". Either within the same ITSM request or by one of our CDWH team members (must have the permission to manage USB Windows AD group memberships).

Software installation

  • Follow instructions at Application install/setup (choice of applications needed on individual basis)
  • User rights and installations of USB applications (like e.g. clinical source applications for DWH like Ismed) have to be requested using Antragsformular: type "antrag" in browser (internet explorer always works)
  • Maybe you have to order more disk space for your VD via SOS-Ticket in ELAS:

    Hint: Service Desk is very thrifty with disk space, if you ask for 40 GB, they will give you 20 GB, maybe. Or your ticket is closed without enlarging the disk, for example when they find files on your local hard drive that they (Service Desk) think you don't need anymore, and they will ask you to clean up first. From experience, it usually needs several tickets until you have the disk space that you need for the software installation!
