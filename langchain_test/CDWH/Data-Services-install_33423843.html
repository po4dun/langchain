DS Access Rights

Before you can start to work with SAP DS, you need your users to be setup in all three repositories. To do so, write an email to Emanuel Madiou and ask him to open a login on all DS servers and assign the repositories to the user. It is always helpful to add an existing user for reference, e.g. Richard Marti (martir). In addition ask the access to the following shared folders:

  • DEV: \\ictsapdaswe01\CDWH
  • KONS: \\ictsapdaswk01\CDWH
  • PROD: \\ictsapdaswp02\CDWH

HANA Remark

If you get an ODBC error message when opening a HANA-Repository, do:

  • Execute %LINK_DIR%\bin\ODBCDriverSelector.exe as Administrator
  • For HANA 2.x ODBC Driver, choose: HDBODBC from the dropdown list
  • Press OK


Windows 10 Remark

If SAP DS Designer not able to launch

Please find the workaround steps here: https://answers.sap.com/questions/13010095/not-able-to-launch-data-services-designer-after-su.html

Workaround:

  • Take a backup of %LINK_DIR%\bin\CreateDemoCerts.bat
  • Edit %LINK_DIR%/bin/CreateDemoCerts.bat on Windows and change the value of VALIDITY_DAYS from 10950 to 9125.
  • Take a backup of folder %LINK_DIR%\ssl
  • Remove all the existing files under %LINK_DIR%\ssl\server
  • Open CMD as Administrator
  • Regenerate SSL certificates by running %LINK_DIR%\bin\CreateDemoCerts.bat (as admin)
  • Check output for any errors. If any occurred, address these before reattempting the step above.
  • Run DS
