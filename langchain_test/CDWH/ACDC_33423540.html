DataWarehouse : ACDC

Created by Fabian Franzeck, last modified on März 06, 2023


Project description

Project run by the finance department to do billing optimisation using a NLP-based Software developed by ID Berlin. This relies mostly on data from the healthengine, pdf documents und data from SAP (via ADT). We are involved to fill the gap with data that they can't get from these sources: These gaps were defined as being covered by: 

  • Cdwh.V_DM_FCT_MEDI_APPL
  • Cdwh.V_DM_FCT_TRANSFUSION

in February 2023, they (Verena Kremser) also requested access to P11.NLEI to run their software (previously agreed to give that data only for the training of their algorythm)→ this was approved by Julian Geisshard (Head of controlling)

Dataflow

They have a VM on the rnd server and connect via JDBC to DWHP.

They have a user on DWHE/K/P (will only use P): IF_ACDC (pw in Keepass)

They have following grants on DWHP:

-- FROM CDWH User:
	--GRANT SELECT ON Cdwh.V_DM_FCT_MEDI_APPL TO IF_ACDC;
	--GRANT SELECT ON Cdwh.V_DM_FCT_TRANSFUSION TO IF_ACDC;
GRANT SELECT ON p11.nlei TO IF_ACDC;

-- FROM ACDC User:
GRANT SELECT ON ACDC.V_DM_FCT_MEDI_APPL TO IF_ACDC;
GRANT SELECT ON ACDC.V_DM_FCT_TRANSFUSION TO IF_ACDC;


p11 grants can't be given from CDWH with grant option to ACDC, thus we have to do the granting directly on an object level...
