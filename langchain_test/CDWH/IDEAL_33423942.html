Project description

IDEAL is a SPHN project with the goal of patient ID linkage from nationwide cohort studies to internal hospital IDs (https://ideal-project.ch/). For this, they plan to run a client in every participating hospital, which has access to the patient directory. On the other hand, it receives tables of subjects included in nationwide cohort studies from the cohort datacenters. On the local client in the hospital, the data will be matched, so a congruence table with cohort IDs and hospital IDs can be created. Those in turn can be used to query, e.g. laboratory data from hospital databases, in order to complete datasets of the cohorts. However, IDEAL is not the agent for the final data transfers of e.g. laboratory data.

The project has a contract with Bram Stieltjes and pay e.g. for a VD on the research server and access to CDWH.

Description of dataflow


DWHP Database schemas for this study:

  • IDEAL has access exclusively to schema IF_IDEAL on DWHP
  • CDWH manages the schema IDEAL on DWHP, which grants SELECT on IDEAL.V_PATIENTS to IF_IDEAL

In Schema IDEAL, we provided one view V_PATIENTS, which contains the following columns (from IL_DIM_PATIENT and IL_DIM_CONSENT). Regarding "Generalkonsent", they are interested in the latest decision recorded by each patient.


DWH_PATIENT_INFO_UPDATE_TS

DWH_CONSENT_INFO_UPDATE_TS

CARE_CENTER_PATIENT_ID

LAST_NAME

FIRST_NAME

DATE_OF_BIRTH

GENDER

HOME_ADDRESS

HOME_POST_CODE

HOME_CITY

LANDLINE_PHONE

MOBILE_PHONE

GENERAL_CONSENT_DECISION

GENERAL_CONSENT_DECISION_DATE


User IF_IDEAL has a SELECT grant to read this view. No other services are provided from CDWH. The client of IDEAL runs on the research server (Andy Aeberhard).
